# 어디보자 — iM뱅크 고령자 인앱 음성 안내 에이전트

> 대신 눌러주지 않고, 옆에서 알려줍니다.
> 마이크가 항상 켜져 어르신의 혼잣말까지 듣고 그때그때 안내하는 인앱 음성 안내 에이전트 프로토타입.

**[iM금융그룹] 제4회 대한민국 청년 인재양성 프로젝트** 출품작 데모입니다.

## 핵심 컨셉
- **타깃**: 창구는 쓰지만 모바일뱅킹 조작이 어려운 고령 사용자
- **원칙**: 실제 거래를 *대신 실행하지 않고*, 다음에 누를 곳을 **안내만** 합니다. 거래가 맞는지 판단은 사용자에게 남깁니다.
- **차별점**: "이게 뭐여", "몰것다" 같은 **혼잣말**까지 인식해 반응하고, 불안(안심 케어)에는 보증 대신 "직접 확인" 을 돕습니다. 도움 횟수가 줄어드는 **자립 곡선**을 지향합니다.

## 화면
**앱 진입 시퀀스**(스플래시 → 지문 인증 → 홈)로 실제 앱처럼 시작하고, iM뱅크 송금 6단계(홈 → 받는사람 → 계좌입력 → 금액 → 확인 → 완료)를 실제 앱 톤으로 재현합니다. 그 위에 어디보자 오버레이(스포트라이트·안내 말풍선·상시 대기 우디 캐릭터)를 얹었습니다.

## 사용법
- 접속하면 **스플래시 → 지문 인식 → 홈** 순으로 진입합니다. (화면을 누르면 건너뜀. 다시 보려면 시나리오 시트의 "스플래시부터 다시 보기")
- 우하단 **우디 버튼**을 누르면 실제 마이크로 혼잣말을 인식합니다. (Chrome 권장)
- 마이크가 없으면 좌하단 **"우디가 듣고 있어요 ›"** 배지를 눌러 혼잣말 시나리오를 체험할 수 있습니다.

## 2단(2-tier) 처리 구조 · 실시간성
실서비스 로직 그대로, **AI 모드는 2단 분기**로 동작합니다.
- **1단 경량(규칙)**: 자주 나오는 표현("이게 뭐여", "됐다 아이가" 등)은 LLM을 거치지 않고 규칙으로 **즉시(수 ms)** 응답 → 비용·지연 최소화.
- **2단 LLM**: 규칙에 안 걸리는 **애매한 발화만** LLM을 호출해 이해 → 정확도 확보.
- **응답 시간 가시화**: 인식→분류까지 걸린 시간을 말풍선에 `⏱ 응답 …` 배지로 표시. 경량은 수 ms, LLM 경로는 통상 1~2초 → 발표에서 "실시간으로 반응한다"를 실제 수치로 증명.
- **파이프라인 표시**: `음성 인식 → 경량 처리 / LLM 호출 → 화면 맥락 → 안내 생성` 4단계 중 2단계 라벨이 경로에 따라 바뀌고, 경량은 초록 즉시 완료·LLM은 파란 호출 대기로 구분됩니다.

## AI 이해 모드 (LLM)
"정해둔 문장만 알아듣는 규칙 기반"을 넘어, **실제 LLM이 혼잣말을 이해**하도록 두 가지 모드를 제공합니다.
- **AI(LLM) 모드**: 인식된 혼잣말 + 현재 화면 단계를 LLM에 보내 5유형(정보부족·혼란·실수·불안·완료)으로 분류하고, 상황에 맞는 안내 문구를 JSON으로 받아 코치마크를 띄웁니다. → *정의하지 않은 발화*("어 이거 참 뭐라 해야 하나", "손이 자꾸 헛짚네")도 이해합니다. 이것이 규칙 기반 데모와의 결정적 차이입니다.
- **규칙 기반 모드**: 인터넷이 불안정한 발표 환경 대비 폴백. LLM 호출이 실패·지연되면 자동으로 규칙 매칭으로 안내가 이어져 데모가 죽지 않습니다.
- **안전 가드**: LLM 프롬프트에 "안내만 한다 · 거래를 대신 실행하지 않는다 · 거래가 맞다고 보증하지 않는다 · 최종 판단은 사용자 몫"을 명시. 경상도 사투리 예시도 포함.
- **API 키**: 코드에 하드코딩하지 않습니다. 발표자가 시트의 "AI 이해 모드"에서 직접 입력하며, 키는 **이 브라우저의 localStorage에만** 저장되고 서버로 전송·커밋되지 않습니다. (Claude / OpenAI 선택, 발표 후 삭제 버튼 제공)

## 완성도 (실서비스 흐름)
- **처리 파이프라인 시각화**: 혼잣말이 들어오면 말풍선 안에 `음성 인식 → AI 이해·보정 → 화면 맥락 → 안내 생성` 4단계가 순차로 채워졌다 사라져, "이게 4단계로 처리된다"를 담백하게 증명합니다. (규칙 모드에선 2단계 라벨이 "표현 매칭")
- **대기 표시**: LLM 응답 전 우디가 "잠깐만요, 지금 화면을 볼게요…"로 침묵 공백을 메우고, 응답이 오면 안내로 전환합니다.
- **어르신 배려 TTS**: 조금 더 천천히(rate 0.88), 긴 문장은 자연스러운 지점에서 끊어 읽기. 연속 발화 시 이전 안내를 정리하고 새 안내로 전환.
- **엣지 케이스**: 못 알아들은 말은 엉뚱한 안내 대신 "한 번만 더 말씀해 주시겠어요?"로 되묻고, 같은 어려움을 반복하면 더 구체적으로 짚는 2차 멘트("아직 헷갈리시죠? …")를 냅니다. 완료 멘트는 이번 세션 실제 개입 횟수를 반영합니다.

## 기술
- 단일 HTML 파일 (HTML + CSS + JS, 외부 의존성 0, 자산은 base64 인라인)
- 의도 파악: LLM API(Claude Messages / OpenAI Chat) 실시간 호출 + 규칙 기반 폴백 토글
- 음성 인식: Web Speech API (`webkitSpeechRecognition`, `ko-KR`)
- 음성 출력: SpeechSynthesis (`ko-KR`)
- 색상: iM 공식 컬러 시스템 (iM Mint `#00C7A9`, iM Lime `#E2F15E` 등)
- 캐릭터/심볼: IM FRIENDS 우디, iM 심볼
- 서체: **Pretendard** — 데모 전체(iM뱅크 앱 목업 + 어디보자 오버레이) 공통 UI 서체. 실제 iM뱅크 앱 화면의 고딕과 가장 근접하며 모든 기기에서 동일하게 렌더. 서브셋 woff2 base64 임베드, 폴백 `'Apple SD Gothic Neo','Malgun Gothic','Noto Sans KR', sans-serif`.
