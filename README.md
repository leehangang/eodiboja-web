# 어디보자 — iM뱅크 고령자 인앱 음성 안내 에이전트

> 대신 눌러주지 않고, 옆에서 알려줍니다.
> 마이크가 항상 켜져 어르신의 혼잣말까지 듣고 그때그때 안내하는 인앱 음성 안내 에이전트 프로토타입.

**[iM금융그룹] 제4회 대한민국 청년 인재양성 프로젝트** 출품작 데모입니다.

## 핵심 컨셉
- **타깃**: 창구는 쓰지만 모바일뱅킹 조작이 어려운 고령 사용자
- **원칙**: 실제 거래를 *대신 실행하지 않고*, 다음에 누를 곳을 **안내만** 합니다. 거래가 맞는지 판단은 사용자에게 남깁니다.
- **차별점**: "이게 뭐여", "몰것다" 같은 **혼잣말**까지 인식해 반응하고, 불안(안심 케어)에는 보증 대신 "직접 확인" 을 돕습니다. 도움 횟수가 줄어드는 **자립 곡선**을 지향합니다.

## 화면
**앱 진입 시퀀스**(스플래시 → 지문 인증 → 홈)로 실제 앱처럼 시작하고, iM뱅크 송금 6단계(홈 → 받는사람 → 계좌입력 → 금액 → 확인 → 완료)를 실제 앱 톤으로 재현합니다. 그 위에 어디보자 오버레이(스포트라이트·안내 말풍선·상시 대기 우디 캐릭터)를 얹었습니다.

## 사용법
- 접속하면 **스플래시 → 지문 인식 → 홈** 순으로 진입합니다. (화면을 누르면 건너뜀. 다시 보려면 시나리오 시트의 "스플래시부터 다시 보기")
- 우하단 **우디 버튼**을 누르면 실제 마이크로 혼잣말을 인식합니다. (Chrome 권장)
- 마이크가 없으면 좌하단 **"우디가 듣고 있어요 ›"** 배지를 눌러 혼잣말 시나리오를 체험할 수 있습니다.

## 2단(2-tier) 처리 구조 · 실시간성
실서비스 로직 그대로, **AI 모드는 2단 분기**로 동작합니다.
- **1단 경량(규칙)**: 자주 나오는 표현("이게 뭐여", "됐다 아이가" 등)은 LLM을 거치지 않고 규칙으로 **즉시(수 ms)** 응답 → 비용·지연 최소화. 말풍선에 **`⚡ 경량 처리 · 즉시 응답`**.
- **2단 AI 정밀 분석**: 규칙으로 확실히 못 잡는 **애매한 발화만** LLM을 호출해 이해. 말풍선에 **`🧠 AI 정밀 분석`**.
- **응답 시간 가시화**: 인식→분류까지 걸린 시간을 말풍선에 `⏱ 응답 …` 배지로 표시. 경량은 수 ms, AI 정밀 분석은 통상 1~2초 → "실시간으로 반응한다"를 실제 수치로 증명.
- **파이프라인 표시**: `음성 인식 → 경량 처리 / AI 정밀 분석 → 화면 맥락 → 안내 생성` 4단계 중 2단계 라벨이 경로에 따라 바뀌고, 경량은 초록 즉시 완료·AI 경로는 파란 분석 대기로 구분됩니다. → "정의된 표현은 빠르게, 처음 보는 표현만 AI로"가 눈에 보입니다.
- 기존 **규칙/AI 수동 토글은 유지**(발표 안정성용). 2단 분기는 AI 모드 안에서만 동작합니다.

## AI 이해 모드 (LLM)
"정해둔 문장만 알아듣는 규칙 기반"을 넘어, **실제 LLM이 혼잣말을 이해**하도록 두 가지 모드를 제공합니다.
- **AI(LLM) 모드**: 인식된 혼잣말 + 현재 화면 단계를 LLM에 보내 5유형(정보부족·혼란·실수·불안·완료)으로 분류하고, 상황에 맞는 안내 문구를 JSON으로 받아 코치마크를 띄웁니다. → *정의하지 않은 발화*("어 이거 참 뭐라 해야 하나", "손이 자꾸 헛짚네")도 이해합니다. 이것이 규칙 기반 데모와의 결정적 차이입니다.
- **규칙 기반 모드**: 인터넷이 불안정한 발표 환경 대비 폴백. LLM 호출이 실패·지연되면 자동으로 규칙 매칭으로 안내가 이어져 데모가 죽지 않습니다.
- **안전 가드**: LLM 프롬프트에 "안내만 한다 · 거래를 대신 실행하지 않는다 · 거래가 맞다고 보증하지 않는다 · 최종 판단은 사용자 몫"을 명시. 경상도 사투리 예시도 포함.
- **API 키**: 코드에 하드코딩하지 않습니다. 발표자가 시트의 "AI 이해 모드"에서 직접 입력하며, 키는 **이 브라우저의 localStorage에만** 저장되고 서버로 전송·커밋되지 않습니다. (Claude / OpenAI 선택, 발표 후 삭제 버튼 제공)

## 발표 자동 시연 (영상 녹화용)
좌측 발표 도구 패널 또는 혼잣말 시트의 **`▶ 발표 자동 시연`** 버튼을 누르면, 골든 시나리오가 **손 안 대고 약 30초간 자동 진행**됩니다 — 진입은 건너뛰고 이체 시작 → "계좌번호가 뭐였더라"(⚡경량) → 금액 화면 "이게 뭐여" → 5만원 입력 → 처음 보는 말(🧠 AI 정밀 분석, 키 있을 때) → 확인 화면 빨간 경고 "와 빨갛노"(안심 케어) → 완료 + 자립 곡선 → "됐다 다 했어"(개입 횟수 반영). 각 혼잣말은 실제 처리 파이프라인·코치마크·TTS를 그대로 발동하므로, **화면만 녹화하면 시연 영상이 됩니다.** 상단 "■ 자동 시연 중지"로 언제든 중단. (API 키가 없으면 '처음 보는 말' 구간은 규칙으로 잡히는 사투리로 자동 대체되어 오프라인에서도 끊기지 않음)

## 현재 화면 맥락 안내 (화면을 강제로 이동하지 않음)
- 어르신이 어느 화면에서 혼잣말을 하든 **그 화면을 벗어나지 않고, 지금 보고 있는 화면에서** 코치마크와 안내를 띄웁니다. (유형=어떻게 도울지, 현재 단계=어디를 짚을지) 예: 금액 화면에서 "이게 뭐여" → 금액 숫자판을 짚고 "5만원은 5·0·0·0·0" 안내. 계좌 화면에서 같은 말 → 계좌번호 칸을 안내.
- **TTS 에코 억제**: 폰 스피커로 나온 우디의 안내 음성을 마이크가 되받아 오인식하는 되먹임을 막기 위해, 우디가 방금 말한 내용과 겹치는 인식 결과는 무시합니다. (조용한 곳/이어폰 권장)

## 이중 입력 안내 (말하거나 누르거나)
- 음성 인식이 **반복 실패(되묻기 2회 이상 연속)** 할 때만 부드럽게 안내: *"말로 하기 어려우시면, 화면을 직접 천천히 눌러도 괜찮아요."* 매번 말하지 않아 잔소리처럼 느껴지지 않게 했고, 한 번이라도 인식에 성공하면 카운터가 초기화됩니다.

## 안내 문구 금액 표기(구어체)
- 화면 UI 숫자(`50,000원`)는 그대로 두고, **음성/안내 문구에서는 구어체**로 말합니다 — "50,000원" 대신 **"5만원"**, "1,250,000원" 대신 "125만원". 어르신께 더 자연스럽게.

## 맥락 상기(context recall) — 작업 도중 목적을 잊을 때
- 어르신이 이체 도중 **"내가 뭐 하려고 했지", "이거 뭐였지", "뭐 하는 중이었노"**(사투리 포함) 같은 방향 상실 발화를 하면, **지금 진행 중인 작업과 단계를 다시 짚어**줍니다. 예) "손자 김민수 님께 50,000원 보내는 중이셨어요. 지금은 금액을 넣는 단계예요. 천천히 이어서 하셔요."
- 현재 화면 단계(screenContext) + 이체 정보(받는사람·금액)를 **앱 상태에서 직접 조합**해 생성(LLM 추측 아님) → 정확. 코치마크는 부드러운 펄스로 지금 위치를 짚어줍니다.
- "방향상실" 유형을 6번째로 추가(규칙 키워드 + LLM 분류 모두). 정의 안 한 방향상실 발화도 AI가 잡아냅니다.

## 확신도 기반 확인(confirmation) — 오인식 방지
- 혼잣말을 인식했는데 **유형 판단이 애매하면**(확신도 낮음) 바로 안내하지 않고 **짧게 확인**부터: 예) "지금 화면이 헷갈리신다는 말씀이시죠?"(혼란), "다시 하고 싶으시다는 말씀이시죠?"(실수). 로봇 같은 긴 오류 문구는 쓰지 않습니다.
- 사용자가 **"응/맞아"** 하면 그 유형으로 안내 진행, **"아니"** 하면 다시 듣기. (응/아니 버튼도 제공 → 마이크 없이 시연)
- 확신도 판단: **규칙 모드** = 키워드가 여러 유형에 걸치면 애매, **AI 모드** = LLM이 `confidence: high|low`를 반환해 낮을 때만 확인. **확실할 땐 확인 없이 바로** 안내(매번 확인하면 답답하므로 애매할 때만).

## 끼어들기(barge-in) — 고령자 음성 UX 핵심
- 우디가 TTS로 안내하는 **도중에 사용자가 말을 시작하면 즉시 안내를 멈추고**(`speechSynthesis.cancel()`) 듣기 모드로 전환. 어르신이 안내를 끝까지 안 기다려도 바로 반응합니다.
- **추임새는 무시**: "어·응·그래·음·네·아" 같은 짧은 맞장구는 멈추지 않고, 추임새가 아닌 실제 단어이거나 일정 길이 이상일 때만 끼어들기로 간주(반복 추임새 "그래그래"도 무시). → 안내 들으며 맞장구치는 습관에도 오작동 안 함.
- 멈추면 **"🔇 안내를 멈추고 듣고 있어요"** 표시가 잠깐 떠서 "말하면 바로 멈춘다"가 눈에 보임. 마이크 없이도 시트의 **끼어들기/추임새 데모 버튼**으로 시연 가능.

## 완성도 (실서비스 흐름)
- **처리 파이프라인 시각화**: 혼잣말이 들어오면 말풍선 안에 `음성 인식 → AI 이해·보정 → 화면 맥락 → 안내 생성` 4단계가 순차로 채워졌다 사라져, "이게 4단계로 처리된다"를 담백하게 증명합니다. (규칙 모드에선 2단계 라벨이 "표현 매칭")
- **대기 표시**: LLM 응답 전 우디가 "잠깐만요, 지금 화면을 볼게요…"로 침묵 공백을 메우고, 응답이 오면 안내로 전환합니다.
- **어르신 배려 TTS**: 조금 더 천천히(rate 0.88), 긴 문장은 자연스러운 지점에서 끊어 읽기. 연속 발화 시 이전 안내를 정리하고 새 안내로 전환.
- **엣지 케이스**: 못 알아들은 말은 엉뚱한 안내 대신 "한 번만 더 말씀해 주시겠어요?"로 되묻고, 같은 어려움을 반복하면 더 구체적으로 짚는 2차 멘트("아직 헷갈리시죠? …")를 냅니다. 완료 멘트는 이번 세션 실제 개입 횟수를 반영합니다.

## 기술
- 단일 HTML 파일 (HTML + CSS + JS, 외부 의존성 0, 자산은 base64 인라인)
- 의도 파악: LLM API(Claude Messages / OpenAI Chat) 실시간 호출 + 규칙 기반 폴백 토글
- 음성 인식: Web Speech API (`webkitSpeechRecognition`, `ko-KR`)
- 음성 출력: SpeechSynthesis (`ko-KR`)
- 색상: iM 공식 컬러 시스템 (iM Mint `#00C7A9`, iM Lime `#E2F15E` 등)
- 캐릭터/심볼: IM FRIENDS 우디, iM 심볼
- 서체: **Pretendard** — 데모 전체(iM뱅크 앱 목업 + 어디보자 오버레이) 공통 UI 서체. 실제 iM뱅크 앱 화면의 고딕과 가장 근접하며 모든 기기에서 동일하게 렌더. 서브셋 woff2 base64 임베드, 폴백 `'Apple SD Gothic Neo','Malgun Gothic','Noto Sans KR', sans-serif`.
